How to Manage Freelance Clients in Notion (Complete Guide)

What You Need to Track (and Why)

Most freelancers either track too little (just names and emails) or too much (every note, every email, every coffee). The sweet spot:

  • Client info — name, email, company, billing rate
  • Status — active, lead, inactive, VIP
  • Projects — linked to clients, with deadlines and budgets
  • Invoices — linked to projects, with payment status
  • Pipeline — prospects moving toward becoming clients

These 5 databases give you a complete picture of your business without drowning in data.

Step-by-Step Setup

1

Create the Client Database

Start a new Notion page and add a database. Key properties to add:

  • Name — Title (default)
  • Email — Email property
  • Company — Text
  • Status — Select: Active, Lead, Inactive, VIP
  • Rate (€/h) — Number, formatted as currency
  • Since — Date (when they became a client)
  • Notes — Text (payment habits, preferred communication)

Add views: Table view (all clients) + filtered "Active Clients" view + "Leads" view.

2

Add the Projects Database

Create a second database with:

  • Project Name — Title
  • Client — Relation (linked to Client database)
  • Status — Select: Not Started, In Progress, Review, Done, On Hold
  • Budget (€) — Number
  • Deadline — Date
  • Progress (%) — Number, formatted as percent

Views: Kanban by Status, Table view, filtered "Active Projects" view.

3

Build the Invoice Tracker

Third database:

  • Invoice # — Title (e.g. INV-2026-001)
  • Client — Relation
  • Amount (€) — Number
  • Status — Select: Draft, Sent, Paid, Overdue
  • Issue Date and Due Date — Date

Add a filtered "Outstanding" view showing Sent + Overdue. Check this every Friday.

4

Create the Sales Pipeline

For managing leads (future clients):

  • Lead Name — Title
  • Stage — Select: New Lead, Contacted, Proposal Sent, Negotiation, Closed Won, Closed Lost
  • Value (€) — Number (estimated project value)
  • Source — Select: Referral, LinkedIn, Website, etc.
  • Follow-up Date — Date
  • Next Action — Text

View: Kanban by Stage. Move leads through the pipeline. Review weekly.

5

Build the Dashboard

Create a master page with linked views of each database. Add a few key metrics at the top:

  • Active clients count
  • Projects in progress
  • Outstanding invoices total
  • Pipeline value

This page becomes your Monday morning ritual — open it, update it, plan your week.

The Weekly Review Process (15 min)

Having the system means nothing without a ritual. Every Monday at 9am:

  1. Check Invoices database — any overdue? Follow up same day.
  2. Update project progress percentages
  3. Move pipeline leads forward (or close the ones going nowhere)
  4. Identify your #1 priority client for the week

Common Mistakes to Avoid

  • Over-engineering from day one — Start simple. Add complexity when you feel the need.
  • Not using relations — Connecting clients → projects → invoices is what makes Notion powerful. Flat databases are just digital paper.
  • Inconsistent status updates — Update status every time something changes. Stale data is worse than no data.
  • Too many views — 2-3 views per database is enough. More becomes noise.
